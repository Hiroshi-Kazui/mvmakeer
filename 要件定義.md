# MV Maker 要件定義書

- 作成日: 2026-07-10
- ステータス: ドラフト(ユーザーレビュー待ち)
- 対象バージョン: v1

---

## 1. 概要

画像と音声ファイルからスライドショー形式のミュージックビデオを作成するデスクトップツール。
曲を再生しながら任意のタイミングでテロップ(歌詞等)を打ち込み、フェードイン/アウトの
タイミングを調整したうえで、テロップ焼き込み済みの mp4 を書き出す。

### 1.1 コアとなる操作体験

1. 画像(複数)と音声(1曲)を選択すると、スライドショーとして即プレビュー再生できる
2. 再生しながら **挿入ボタン(`I`キー)** を押すと、その再生位置にテロップが挿入され「アクティブ」になる
3. アクティブなテロップに対して **フェードアウトボタン(`O`キー)** を押すと、その再生位置がフェードアウト開始点になる
4. 挿入済みテロップのフェードイン/フェードアウト位置は、タイムライン上のドラッグまたは数値入力で後から編集できる
5. 最後に mp4 として書き出す

## 2. 技術方針

| 項目 | 決定 | 理由 |
|---|---|---|
| フレームワーク | **Tauri 2 + TypeScript + Vite**(フロントは素のTS) | `C:\develop\週日の集会割当` と同一スタック。Win/Mac 両対応、ツールチェーン確立済み |
| プレビュー | **ブラウザ側でリアルタイム合成**(画像切替 + `<audio>` 再生 + テロップDOMオーバーレイ) | 中間動画ファイルを生成しない。編集が即座に反映され、生成待ちが発生しない |
| エンコード | **ffmpeg をサイドカーバイナリとして同梱**し、最終書き出し時のみ実行 | スライドショー合成もテロップ焼き込みも ffmpeg 一本で完結 |
| 対応OS | v1: Windows。macOS は将来対応(コードは最初からOS依存を避ける) | |
| UIベース | `mocks/mvmaker-ui.html` のレイアウト・配色を踏襲 | ただし歌詞キューは廃止(§4.4) |

## 3. 利用者・配布

- 当面は作者本人専用。ただし将来配布する可能性があるため:
  - エラーはユーザーに読める形で表示する(コンソールのみに出さない)
  - 設定・プロジェクトファイルの保存先はOS標準の場所に従う
  - v1 ではインストーラ・コード署名・自動更新はスコープ外

## 4. 機能要件

### 4.1 素材管理

- **画像**: jpg / png を複数選択で追加できる。サムネイル付き一覧に表示
- 一覧はドラッグ&ドロップで並べ替え可能(表示順 = 再生順)
- 画像の削除・追加はいつでも可能
- **音声**: mp3 / wav / m4a を1ファイル選択できる。差し替え可能
- 動画全体の長さ = 音声ファイルの長さ

### 4.2 スライドショー

- 各画像の表示時間は、初期状態では **均等割り**(音声の長さ ÷ 枚数)
- 各画像の表示時間は **後から個別に編集可能**(1枚の秒数を変えたら、残りの未固定画像で残時間を再配分する)
- 切替エフェクトは **クロスフェード固定**(時間は内部固定値、例: 1.0秒)
- 解像度・アスペクト比は **1920×1080 (16:9) / 30fps 固定**。画像はアスペクト維持で
  レターボックス(黒帯)またはカバー(はみ出しクロップ)— 既定はカバー

### 4.3 プレビュー再生

- 再生 / 一時停止(`Space`)、先頭へ、シークバー、`←`/`→` で ±1秒シーク
- 現在時刻 / 総時間のタイムコード表示(mm:ss.d)
- プレビューはリアルタイム合成のため、素材・テロップの変更が即座に反映される

### 4.4 テロップ

#### 挿入
- 再生中・停止中を問わず、**挿入ボタンまたは `I` キー**で現在の再生位置にテロップを挿入
- **テキストは挿入後にその場で入力する**(事前登録の歌詞キューは採用しない)。
  挿入直後、右パネルのテキスト欄にフォーカスが移る。再生は止めない(ユーザーが任意で停止)
- 挿入直後のテロップは「アクティブ」状態になり、プレビュー上にバッジ表示

#### フェード
- 挿入位置 = フェードイン開始位置
- アクティブなテロップに対し **フェードアウトボタンまたは `O` キー**で、現在の再生位置を
  フェードアウト開始位置に設定。テロップ終了 = フェードアウト開始 + フェードアウト時間
- フェード時間デフォルト: **イン 0.5秒 / アウト 0.8秒**(テロップごとに変更可能)
- フェードイン開始 / フェードアウト開始 / 終了位置は、タイムライン上のドラッグと
  数値ステッパー(±0.1秒)の両方で後から編集可能

#### スタイル(テロップごとに個別編集可能)
- テキスト(複数行可)
- フォント(システムフォントから選択)
- サイズ
- 文字色
- 画面内位置(9分割グリッドから選択)
- 影は固定で常時付与(縁取りは不要)

#### 同時表示
- v1 では **同時に表示されるテロップは1つ**。時間帯が重なる挿入をした場合は、
  前のテロップのフェードアウトを新テロップの開始位置に自動で詰める
- 複数テロップの重ね表示は将来対応(スコープ外、§7)

### 4.5 タイムライン

- 表示トラック: テロップ / 画像(各画像の区間) / 音声波形
- **音声波形を表示する**(打ち込みタイミングの目安)
- 再生位置プレイヘッド表示。ルーラーのクリック / ドラッグでシーク
- テロップクリップ: クリックで選択(=アクティブ化)、ドラッグで移動、
  左右端とフェードアウトマーカーのドラッグで各位置を編集
- 画像区間: 境界ドラッグで表示時間を編集(§4.2)

### 4.6 プロジェクト保存・再開

- プロジェクトをファイルに保存し、後日開いて編集を再開できる
- 形式: JSON 1ファイル(拡張子 `.mvproj`)。素材は**絶対パス参照**(埋め込まない)
- 開いたとき素材ファイルが見つからない場合は、その旨を表示して再指定を促す
- 構造(サンプル値は架空):

```json
{
  "version": 1,
  "name": "My Music Video",
  "audio": { "path": "C:/media/song.mp3", "duration": 225.0 },
  "images": [
    { "path": "C:/media/img_001.jpg", "duration": 45.0 }
  ],
  "telops": [
    {
      "id": 1,
      "text": "この景色が\n心をつなぐ",
      "timeIn": 75.2,
      "fadeInDur": 0.5,
      "fadeOutStart": 79.8,
      "fadeOutDur": 0.8,
      "style": { "font": "游明朝", "size": 72, "color": "#ffffff", "position": "center" }
    }
  ]
}
```

### 4.7 書き出し

- 出力: **mp4(H.264 + AAC)、1920×1080、30fps、テロップ焼き込み済み**
- 実装: ffmpeg サイドカーを1回実行
  - 画像列 → クロスフェード合成(`xfade`)
  - テロップ → `ass` 字幕(フェードはASSタグ)を焼き込み、または `drawtext`(実装時に選定)
  - 音声をそのまま多重化(再エンコードは必要時のみ)
- 進捗バーを表示し、キャンセル可能
- 完了後、出力先フォルダを開くボタンを表示

## 5. 非機能要件

- プレビューの再生位置とテロップ表示のずれは、打ち込み用途に耐える精度(±0.1秒以内目安)とする
- 想定規模: 画像 ~50枚、テロップ ~100個、曲長 ~10分で快適に動作すること
- 破壊的操作(テロップ削除、素材削除、未保存で閉じる)は確認ダイアログを出す
- キーボードショートカット: `Space` 再生/停止、`I` 挿入、`O` フェードアウト、`←`/`→` シーク、`Del` 削除

## 6. 画面構成

`mocks/mvmaker-ui.html` を基本とする(3カラム+タイムライン):

- **左パネル**: 素材(画像一覧・音声)。※モックにあった「歌詞キュー」は廃止
- **中央**: プレビュー / トランスポート / 挿入・フェードアウト・削除ボタン / タイムライン
- **右パネル**: 選択中テロップのインスペクタ(テキスト、フォント、サイズ、色、位置、タイミング)
- **フッター**: ショートカットガイド

## 7. スコープ外(v1 では実装しない)

| 項目 | 備考 |
|---|---|
| テロップの同時複数表示 | データ構造は重なりを許容する形にしておき、将来対応しやすくする |
| Undo / Redo | 削除の確認ダイアログで代替 |
| 縦動画・解像度選択 | 1080p 16:9 固定 |
| 切替エフェクトの選択 | クロスフェード固定 |
| テロップの縁取り・アニメーション種別 | フェードのみ、影固定 |
| インストーラ・署名・自動更新 | 配布時に別途検討 |
| macOS ビルド | コードはOS非依存に保つが、ビルド・検証は将来 |

## 8. 未決事項(実装時に決定)

- ffmpeg のテロップ焼き込み方式(`ass` vs `drawtext`)— 日本語フォント指定とフェード表現の両立で選定
- 画像の「カバー/レターボックス」切替をUIに出すか、カバー固定にするか
- ffmpeg サイドカーのライセンス表記(GPL構成 or LGPL構成)— 配布時に確定
